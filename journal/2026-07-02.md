## Decisions made

1. **One system, paired items — not two decks.** Every Kali item (black hole / void / dark mass) is joined to a Paradevi counterpart (galaxy / radiant structure) by an `emanation` cross-reference. Mode is a lens, not a fork.
2. **The emanation link is bidirectional.** This came straight from the corpus work: Wallis's actual framing is that Kali and Para are "two sides of the same coin... aspects of each other" — mutual aspecthood, not a one-way descent. Emanation is the Goddess's own cycle (the Twelve Kalis and Mahakali), so the flip works both ways. See `research/wallis-grounding.md`.
3. **Source found for the central image:** in an archived AMA satsang, Wallis himself used this project's central image — Kali as the black hole, Para as the white hole — years before this repo existed. The design conceit has a source in the teaching it honors. (Caption garble on "Para"; flagged for verification against the video.)
4. **Two data layers, strictly separated.** Astronomy layer (name, catalog id, type, distance, verified fact, credited image) and reflective layer (theme, prompt, shadow/light reading, emanation link). Facts are science; the mirror is labeled as a mirror. This is the honesty stance made structural.
5. **30 objects shortlisted and paired** — 15 Kali, 15 Paradevi, each pair with a one-line reason (`research/objects-shortlist.md`). Anchor pair: Sgr A* ↔ Milky Way, the void at the heart of our own radiance.
6. **Theming reality accepted:** grammars can't force UI dark/light mode today, so mode = content + imagery first. If the platform grows a theming hook, mode gets inferred from data, never stored as a grammar type.
7. **License placeholder:** suggested CC BY-SA 4.0 for data/text (matching the tarot repo); builder decides.
8. **Non-consent axiom enforced in card grammar:** prompts are questions, readings are offered framings, no "you are / you will."
